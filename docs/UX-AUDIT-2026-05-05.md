# UX & Feature Audit — Clarity Space
**Дата:** 2026-05-05
**Сравнение с:** Notion, Linear, Todoist, ClickUp, Monday.com

---

## Что отлично работает

1. **Мульти-вью система** — Kanban/Timeline/Gantt/Calendar — мирового уровня. Пользователь выбирает как смотреть на задачи.
2. **Встречи как first-class** — транскрибация, AI-саммери (Notes + Q&A), выбор задач из встречи, Obsidian sync. Опережает конкурентов.
3. **Мобильный swipe** — TodaySwipePage как Tinder для задач. Свайп вправо = готово. Гениально для быстрого разбора.
4. **AI-ассистент** — голосовые команды, контекст встречи, создание задач/встреч/идей голосом.
5. **Визуальный язык** — consistent тёмная тема, градиенты, анимированные сферы, bilingual (RU/EN).
6. **Дашборд** — AI Daily Plan, виджеты задач/встреч/идей, прогресс-бары проектов.

---

## UX проблемы

### Критичные
1. **Нет кнопки "+ Задача" на Kanban** — пользователь не понимает как создать задачу. Нужна кнопка в заголовке. *15 мин фикс.*
2. **Период и статус смешаны** — в TaskDetailPanel "Сегодня/Завтра" и "В работе/Готово" в одном списке кнопок. Нужно разделить на "Статус" и "Дата". *30 мин фикс.*

### Средние
3. **Timeline drag-and-drop скрыт** — можно перетаскивать задачи между периодами, но нет визуальной подсказки.
4. **Мобильные табы Kanban** — "Бэк" вместо "Бэклог" нечитаемо.
5. **Gantt без легенды** — непонятно что означают цвета плашек.
6. **Нет onboarding** — новый пользователь видит пустую доску без подсказок.
7. **Горячие клавиши не видны** — есть `useHotkeys()` но нет UI с подсказками.

### Мелкие
8. **Пустые состояния** — "Нет задач" вместо "Всё разобрано! Создай задачу или проект".
9. **Просроченные задачи** — только красный текст, нужен бейдж "ПРОСРОЧЕНО".
10. **Telegram бот не рекламируется** — нет ссылки "Используй в Telegram" в интерфейсе.

## Чего не хватает (vs конкуренты)

### vs Notion
- **Inline editing** — нельзя переименовать задачу кликом на карточке, надо открывать панель
- **Database relations** — нет связей задачи→цель, проект→люди визуально
- **Шаблоны блоков** — нет reusable templates для задач/встреч

### vs Linear
- **Циклы/спринты** — нет планирования по спринтам
- **Velocity tracking** — нет burndown/burnup графиков
- **Визуализация зависимостей** — зависимости есть, но нет визуальной цепочки
- **Прогнозирование** — "при текущей скорости закончите через 3 недели"

### vs Todoist/ClickUp
- **Трекинг времени** — нет оценки/логирования часов
- **Bulk operations** — нельзя "отметить все как готово" или "сменить приоритет у 10 задач"
- **Нативное приложение** — только web + TG

### vs Monday.com
- **Ресурсное планирование** — нет "кто перегружен, кто свободен"
- **Автоматизации** — нет "если задача готова → переместить в архив"

## Суперфичи для предпринимателей (TOP-10)

### Приоритет 1 — делать первыми

**1. Projects Dashboard** (4-6 часов)
Каждый проект как карточка: прогресс-бар, следующий milestone, количество просроченных задач, аватарки команды, дата последней встречи. Предприниматель видит с одного взгляда какие проекты в беде.

**2. Quick Capture (Cmd+K)** (8-10 часов)
Из любого места: Cmd+K → модалка → пишешь "Написать статью @Маркетинг !p1 до:пятницы" → задача создана. AI парсит natural language. 10x быстрее создание задач.

**3. Daily Standup One-Pager** (6-8 часов)
Замена дашборда для предпринимателя:
- Вверху: "Что сделали вчера?" (выполненные задачи + revenue impact)
- Середина: "Что блокирует?" (просроченные + зависимости)
- Внизу: "Фокус сегодня?" (AI-приоритизация)

**4. Revenue Tags** (4-5 часов)
Тег на задаче: "$5K", "$100K". Дашборд: "Вы работаете над задачами на $2M выручки". Когда задача выполнена → запись как revenue. Основатели хотят ROI от каждой задачи.

**5. People Velocity** (6-8 часов)
Sidebar: каждый член команды — задачи за неделю, тренд, загруженность. Видно кто перегружен до burnout.

### Приоритет 2 — следующий спринт

**6. Dependency Chain Visualizer** (8-10 часов)
При просмотре задачи: "Эта задача блокирует 5 других" + визуальная цепочка.

**7. Burndown Forecasting** (4-6 часов)
Velocity 5 задач/неделю, осталось 30 → "Завершение ~15 марта". Для investor updates.

**8. Async Standup через TG** (6-8 часов)
Бот спрашивает "Что сделал сегодня?" → пользователь голосом → транскрибирует → отмечает задачи как выполненные → постит в канал.

### Приоритет 3 — polish

**9. Project Health Score** (3-4 часа)
Зелёный/жёлтый/красный индикатор на проекте. Считается: (задачи в срок) / (всего задач).

**10. Export to CSV** (15 мин)
Кнопка "Скопировать в CSV" для вставки в инвесторскую презентацию.

## Позиционирование

**Сильные стороны:**
- Multi-view (Kanban/Timeline/Gantt) — мирового уровня
- AI meeting processing — опережает конкурентов
- Mobile swipe UX — уникален
- Obsidian integration — решает проблему "единого источника правды"

**Слабые стороны:**
- Нет revenue/impact tracking (критично для основателей)
- Нет time tracking и velocity метрик
- Нет видимости загрузки команды

**Конкурентное позиционирование:**
- **vs Notion** — быстрее, opinionated, AI встречи. Notion тормозит.
- **vs Linear** — проще, лучше для high-level планирования. Founders не нужны issue cycles.
- **vs Asana** — 10x дешевле, Telegram mobile, AI processing.

**Вывод:** Фундамент отличный. Добавить revenue tags + velocity chart + people workload → продукт станет #1 для solo/early-stage founders.
